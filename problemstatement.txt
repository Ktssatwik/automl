Problem Statement - Agentic AutoML POC (LLM-Based)
==================================================

Background
----------
Users with domain datasets often cannot manually perform target detection, problem framing, EDA interpretation, and end-to-end ML pipeline design.

Core Problem
------------
Given only a CSV upload, build an agentic AutoML system that automatically:
- identifies the most likely target,
- determines classification vs regression,
- infers business domain,
- performs EDA and downstream ML pipeline steps,
- provides prediction and downloadable model outputs.

Objective
---------
Create a complete POC with a Master Agent and specialist agents where key data-understanding decisions are LLM-based, while maintaining robust fallback behavior.

Scope
-----
In scope:
- CSV upload and profiling
- LLM-based target detection reasoning
- LLM-based problem type detection
- LLM-based domain understanding
- EDA metrics + LLM EDA narrative
- Job-based tracking and API-driven orchestration
- Streamlit UI integration (later phases)

Out of scope (first POC):
- Advanced hyperparameter optimization
- Distributed training infrastructure
- Production authentication and enterprise deployment controls

Functional Requirements
-----------------------
- FastAPI endpoints for upload, run, status, EDA/results, prediction, and model download.
- Multi-agent architecture with one class per agent.
- LLM service wrapper for strict JSON outputs.
- Structured step outputs stored by job_id.
- Fallback logic when LLM is unavailable.

Success Criteria
----------------
- User uploads CSV and gets autonomous pipeline progress.
- System returns target/problem/domain decisions with reasoning.
- `decision_mode` indicates whether output came from LLM or fallback.
- Pipeline remains stable even if LLM is unavailable.
- End-to-end flow is demonstrable and extensible.

Future Extensions
-----------------
- Add MCP tools to enrich agent context.
- Add stronger prompt policies and output validators.
- Expand model search, explainability, and deployment adapters.
