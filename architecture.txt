Agentic AutoML POC - Architecture
================================

1. Goal
- Build a working POC where user uploads a CSV and the system auto-runs complete ML pipeline using coordinated agents.
- Deliver prediction capability and downloadable trained model.

3. Core Runtime Components
- Streamlit frontend:
  - CSV upload
  - Run AutoML button
  - Poll and show pipeline status
  - Show EDA/results/model comparison
  - Prediction input form
  - Model download action

- FastAPI backend:
  - upload/run/status/result/predict/download endpoints
  - validates input and routes work to Master Agent

- Master Agent:
  - maintains context and step status
  - executes specialist agents in strict order
  - captures outputs/errors and final report

- Specialist Agents:
  - one responsibility per file
  - deterministic rule-based logic for POC
  - structured JSON-like return payloads

- Services:
  - persistent job state keyed by job_id
  - model artifact and metadata persistence
  - reusable utility methods

4. Data and Artifact Flow
- Upload CSV -> save to storage/uploads/{job_id}.csv
- Run pipeline -> generate EDA + model metrics + best model
- Save outputs:
  - storage/models/{job_id}_pipeline.joblib
  - storage/models/{job_id}_metadata.json
  - storage/reports/{job_id}_eda.json
  - storage/reports/{job_id}_report.json

5. MCP/LLM Ready Design Points
- Keep agent contract stable: run(context: dict) -> dict
- Add explicit placeholders in each agent for future MCP tool calls
- Keep reasoning fields in outputs so LLM agent replacement is easy

6. POC Boundaries
- No advanced tuning or distributed training
- No production auth/queueing
- Focus on clean modular architecture and full end-to-end usability
