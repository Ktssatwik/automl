Agentic AutoML POC - Architecture
================================

1. Goal
- Build a working POC where user uploads a CSV and the system auto-runs complete ML pipeline using coordinated agents.
- Deliver prediction capability and downloadable trained model.

3. Core Runtime Components
- Master Agent orchestration + specialist agents.
- LLM-first specialist logic in Phase 4 agents:
  - CSVUnderstandingAgent
  - ProblemTypeAgent
  - DomainAgent
  - EDAAgent
- Deterministic fallback heuristics if LLM key/config is unavailable.

4. LLM Architecture (Current POC)
- Agent computes structured dataset metadata with pandas.
- Agent sends compact metadata payload to LLM via `llm_service.py`.
- LLM returns strict JSON decisions/reasoning.
- Agent validates output and stores final structured result.
- If LLM fails, fallback logic prevents pipeline failure.

5. Data and Artifact Flow
- Upload CSV -> storage/uploads/{job_id}.csv
- Run pipeline -> agent outputs + EDA/report artifacts
- Save outputs:
  - storage/reports/{job_id}_eda.json
  - storage/reports/{job_id}_report.json
  - later phases add model artifacts in storage/models/

6. MCP/LLM Integration Direction
- Current: LLM via OpenAI API service wrapper.
- Next: MCP tool hooks can be plugged per agent to enrich context (domain KBs, profiling tools, report tools).

7. POC Boundaries
- No advanced tuning/distributed compute.
- No production auth/queueing.
- Focus on a clear LLM-based agentic AutoML pipeline with end-to-end flow.
