Agentic AutoML POC - Delivery Pipeline (LLM-Based)
=================================================

Phase 1: Project Setup and Skeleton
-----------------------------------
- Create backend/frontend/agents/services/schemas/storage structure.
- Add base dependencies and app entry points.

Phase 2: Core Backend APIs and Job State
----------------------------------------
- Implement upload + job_id lifecycle.
- Implement required API contract endpoints.
- Add pipeline status tracking with step-level state.

Phase 3: Agent Framework and Master Orchestration
--------------------------------------------------
- Create BaseAgent interface.
- Create MasterAgent for ordered execution.
- Add per-step status transitions and error handling.

Phase 4: LLM-Based Data Understanding Agents
--------------------------------------------
What to do:
- Build `llm_service.py` for OpenAI-based JSON reasoning calls.
- CSV Understanding Agent:
  - compute dataset metadata with pandas,
  - send metadata to LLM,
  - get selected target + reasoning.
- Problem Type Agent:
  - use target stats + sample values,
  - ask LLM for classification vs regression decision.
- Domain Agent:
  - send columns + target context,
  - get inferred domain + preprocessing notes from LLM.
- EDA Agent:
  - compute deterministic stats,
  - ask LLM for analytical EDA narrative and cautions.
- Add fallback logic if LLM unavailable.

Done criteria:
- For uploaded CSV, system returns target/problem/domain/EDA with `decision_mode` showing `llm` when configured.
- With no LLM key, pipeline still runs with fallback outputs.

Phase 5: ML Pipeline Agents
---------------------------
- Preprocessing (ColumnTransformer + Pipeline)
- Train/test split
- Multi-model training
- Cross-validation
- Metrics + model ranking
- Best model selection + save

Phase 6: Prediction and Reporting
---------------------------------
- Prediction agent using saved model pipeline.
- Final report agent consolidating all outputs.
- Wire result/predict/download endpoints fully.

Phase 7: Streamlit Integration
------------------------------
- CSV upload, run trigger, step tracker.
- EDA + model results visualization.
- Prediction form and model download.

Phase 8: Testing and Documentation
----------------------------------
- Test on multiple datasets.
- Add sample insurance CSV and README runbook.
- Document LLM setup (`OPENAI_API_KEY`, model env var).

Execution Order
---------------
1. Setup
2. API + state
3. Orchestration
4. LLM data understanding
5. ML training pipeline
6. Prediction/reporting
7. Frontend integration
8. Testing/docs
